# 로또
### 1. 도메인(Model) 설계 및 구현
가장 작은 단위의 객체부터 구현하여 원시값 포장 규칙을 지킵니다.

[X] 로또 번호 객체 (LottoNumber) 구현 (규칙 3: 원시값 포장)
[X] 1 ~ 45 사이의 숫자인지 검증한다.
[X] 동등성 비교를 위해 equals()를 구현한다.

[ ] 로또 객체 (Lotto) 구현 
[X] List<LottoNumber>를 멤버 변수로 가진다.
[X] 번호가 6개인지 검증한다.
[X] 번호가 중복되지 않는지 검증한다.
[X] 특정 번호가 포함되어 있는지 확인하는 contains 메소드를 구현한다.
[ ] 생성 시 내부에서 리스트를 정렬(Collections.sort)한다.

[ ] 당첨 순위 (Rank) Enum 구현
[ ] 1등~5등, 꽝에 대한 매칭 개수와 당첨 금액 정보를 상수로 정의한다.
[ ] 매칭 개수와 보너스 볼 일치 여부를 받아 해당 Rank를 반환하는 valueOf 로직을 구현한다.

[ ] 구입 금액 객체 (Money) 구현 
[ ] 1,000원 미만인지 검증한다.
[ ] 금액에 따라 구매 가능한 로또 장수를 계산하는 기능을 구현한다.
[ ] 총 당첨금을 받아 수익률을 계산하는 기능을 구현한다.

[ ] 구매한 로또 목록 (LottoTickets) 구현.
[ ] List<Lotto>를 멤버 변수로 가진다.
[ ] 로또 개수를 입력받아 자동으로 로또를 생성하는 정적 팩토리 메소드를 구현한다.
[ ] 로또 번호들이 중복되지 않는지 검증한다.
[ ] 당첨 로또와 내가 구핸 로또를 비교해 전체 당첨 결과를 반환하는 기능을 구현한다.

### 2. 입출력 및 컨트롤러 구현
비즈니스 로직과 UI 로직을 분리합니다.
[ ] 구입 금액 입력 메서드 (String 반환 -> Controller에서 Money로 변환)
[ ] 당첨 번호 입력 메서드 (String 반환 -> ,로 분리 및 파싱)
[ ] 보너스 볼 입력 메서드

[ ] 구매 개수 및 구매한 로또 번호 목록 출력 메서드
[ ] 당첨 통계 헤더 출력 메서드
[ ] 각 등수별 당첨 개수 출력 메서드

[ ] 수익률 출력 메서드

[ ] 흐름 제어: 금액 입력 -> 로또 구매 -> 구매 목록 출력 -> 당첨 번호 입력 -> 통계 계산 -> 결과 출력.
