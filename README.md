# 로또
### 1. 도메인(Model) 설계 및 구현
가장 작은 단위의 객체부터 구현하여 원시값 포장 규칙을 지킵니다.

[X] 로또 번호 객체 (LottoNumber) 구현 (규칙 3: 원시값 포장)
[X] 1 ~ 45 사이의 숫자인지 검증한다.
[X] 동등성 비교를 위해 equals()를 구현한다.

[X] 로또 객체 (Lotto) 구현 
[X] List<LottoNumber>를 멤버 변수로 가진다.
[X] 번호가 6개인지 검증한다.
[X] 번호가 중복되지 않는지 검증한다.
[X] 특정 번호가 포함되어 있는지 확인하는 contains 메소드를 구현한다.
[X] 생성 시 내부에서 리스트를 정렬(Collections.sort)한다.

[X] 당첨 순위 (Rank) Enum 구현
[X] 1등~5등, 꽝에 대한 매칭 개수와 당첨 금액 정보를 상수로 정의한다.
[X] 매칭 개수와 보너스 볼 일치 여부를 받아 해당 Rank를 반환하는 valueOf 로직을 구현한다.

[X] 구입 금액 객체 (Money) 구현 
[X] 금액에 따라 구매 가능한 로또 장수를 계산하는 기능을 구현한다.
[X] 총 당첨금을 받아 수익률을 계산하는 기능을 구현한다.

[X] 구매한 로또 목록 (LottoTickets) 구현.
[X] List<Lotto>를 멤버 변수로 가진다.
[X] 로또 개수를 입력받아 자동으로 로또를 생성하는 정적 팩토리 메소드를 구현한다.
[X] 로또 번호들이 중복되지 않는지 검증한다.
[X] 당첨 로또와 내가 구한 로또를 비교해 전체 당첨 결과를 반환하는 기능을 구현한다.

### 2. 입출력 및 컨트롤러 구현
비즈니스 로직과 UI 로직을 분리합니다.

[X] 1,000원 미만인지 검증한다.

[X] 구매 개수 및 구매한 로또 번호 목록 출력 메서드
[X] 당첨 통계 헤더 출력 메서드
[X] 각 등수별 당첨 개수 출력 메서드

[X] 수익률 출력 메서드

[X] 흐름 제어: 금액 입력 -> 로또 구매 -> 구매 목록 출력 -> 당첨 번호 입력 -> 통계 계산 -> 결과 출력.


# 로또 미션 2
### 1. 도메인 로직 
[X] 사용자가 구매할 수 있는 로또의 개수를 확인한다.
[X] 수동 로또 생성: 사용자가 입력한 번호로 로또 티켓을 생성한다.
[X] 수동 로또를 생성 할 때, 번호가 중복되지 않게 확인한다.
[X] 자동 로또 생성: (구입 금액 - 수동 구매 금액)에 해당하는 수량만큼 로또를 자동 생성한다.
[X] 사용자가 구매한 모든 로또(수동+자동)를 당첨 번호와 비교한다.

### 2. 입력 로직 
[ ] 사용자에게 수동으로 구매할 로또 장수를 입력받는다.
[ ] [예외] 입력 값이 숫자가 아닌 경우.
[ ] [예외] 수동 구매 횟수가 음수인 경우.
[ ] [예외] 수동 구매 금액(횟수 × 1,000)이 전체 구입 금액을 초과하는 경우.
[ ] 수동 구매 횟수만큼 로또 번호(6개)를 입력받는다.
[ ] [예외] 번호가 6개가 아닌 경우.
[ ] [예외] 번호가 1~45 범위를 벗어난 경우.
[ ] [예외] 번호 간 중복이 있는 경우.

### 3. 출력 로직
[ ] 수동 구매 개수와 자동 구매 개수를 안내 문구로 출력한다.
[ ] 수동으로 구매한 로또를 먼저 출력하고, 이어서 자동 로또를 출력한다.
[ ] 2등의 경우 "보너스 볼 일치" 문구를 포함한다.
